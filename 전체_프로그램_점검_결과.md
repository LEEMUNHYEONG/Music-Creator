## 2. 확인된 정상 동작

- **MV 프롬프트 생성**: `window.generateSceneOverview` (app.js 6582행) — era, country, location, characters는 함수 상단에서 한 번만 선언 후 썸네일/배경/인물 생성 시 재사용.
- **프로젝트 목록**: loadProjectList, filterProjects, sortProjects, initProjectDragAndDrop, deleteProject, duplicateProject 정상 정의.
- **헤더 버튼**: handleHeaderButtonClick에서 manualBackup 시 확인→전체 백업, 취소→프로젝트만 백업.
- **파일 구조**: app.js(약 13,084행), index.html(약 4,999행), styles.css 존재.

---

## 3. 권장 사항 (선택)

- **정기 백업**: 수정 전에 상단 "백업" 버튼으로 전체 프로그램 백업 권장.
- **브라우저 캐시**: 코드 수정 후 반영이 안 되면 Ctrl+Shift+R(강력 새로고침) 후 재테스트.
- **콘솔 확인**: "함수를 찾을 수 없습니다" 메시지가 나오면 app.js가 완전히 로드되기 전에 버튼을 누른 경우일 수 있으므로, 페이지 로드 후 잠시 뒤 다시 시도.

## 4. 추가 확인

- **중괄호 개수**: app.js에서 `{`와 `}` 개수가 문자열/정규식 포함 시 4개 차이날 수 있음. Linter 기준 문법 오류 없음으로 판단.
